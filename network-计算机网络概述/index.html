计算机网络概述

发表于 2019-11-07 | 更新于 2019-11-20 | 分类于 计算机网络

基本概念

计算机网络

  • 定义：计算机网络就是互连的、自治的计算机集合

    • 自治 - 没有主从关系
    • 互连 - 互联互通
    • 不同的主机通过通信链路进行连接

  • 距离远、数量大时，使用交换网络进行互连

  • Internet：

  • 组成细节角度

    • 全球最大的互联网络

    • ISP(Internet Service Provider)：网络互连的“网络之网络”，也就是“主机”就是网络的网络

    • 数以百万计的互连的计算设备的集合：称作主机(hosts)或端系统，都会运行网络应用，通过通信链路连接在一起

    • 分组交换：转发分组（数据包）

      • 路由器和交换机

  • 服务角度：

    • Internet为网络应用提供通信服务的通信基础设施
    • 为网络应用提供应用编程接口(API)

网络协议

  • 计算机网络中的数据交换必须遵守事先约定好的规则，这些规则就是协议
  • 网络通信：
    • 信息主体是“机器”
    • 交换“电子化”和“数字化”的信息
    • 计算机网络的所有通信过程都必须遵守某种/些规则，这些规则就是协议
  • 网络协议（network protocol）：简称为协议，是为进行网络中的数据交换而建立的规则、标准或约定
    • 协议规定了通信实体之间所交换的信息的格式、意义、顺序以及针对收到信息或发生的时间所采取的动作
  • 协议的三要素：
    • 语法（Syntax）
      • 数据与控制信息的结构或格式
      • 信号电平
    • 语义（Semantics）
      • 需要发出何种控制信息
      • 完成何种动作以及做出何种响应
      • 差错控制
    • 时序（Timing）
      • 时间顺序
      • 速度匹配
  • Internet协议标准：RFC、IETF

通俗地讲，协议就是两台计算机之间通信索要遵守的共同标准，比如一个广东人和一个四川人想要交流，就可以使用双方都懂的普通话，这时“普通话”就可以看作一种“协议”。而网络协议也是为了保证不同计算设备能够正常交流所制定的规则。

计算机网络结构

  • 网络边缘：
    • 主机、网络应用
    • 通过物理介质接入网络，如有线或无线通信链路
  • 网络核心（核心网络）
    • 互联的路由器（或分组转发设备）
    • 网络的网络
  • 主机（端系统）：
    • 位于网络的最边缘处，会运行各种网络应用程序，如：Web，email
    • 客户/服务器应用模型（client/server）：
      • 客户发送请求，接收服务器响应
      • 如：Web应用，文件传输FTP应用
    • 对等应用模型（peer-peer，P2P）：
      • 无（或不仅以来）专用服务器
      • 通信在对等实体之间直接进行
      • 如：Gnutelia，BT，QQ

  • 接入网络 ：

    • 将网络边缘接入核心网（边缘路由器）

    • 用户最关心的是：

      • 带宽（bandwidth）（bps）：与其他领域的带宽不同（单位为Hz）
      • 共享/独有

    • 数字用户线路（DSL）：

      • 利用已有的电话线连接中心局的DSLAM

        • 数据通信通过DSL电话线接入Internet
        • 语音通过DSL电话线接入Internet

      • <2.5Mbps上行传输速率（典型速率 < 1 Mbps）（上传）

      • <24Mbps下行传输速率（典型速率 < 10 Mbps）（下载）

      • FDM: > 50kHz - 1 MHz 用于下行

        ​ 4kHz - 50kHz用于上行

        ​ 0kHz - 4kHz用于传统电话

    • 电缆网络：使用频分多路复用技术，在不同频带（载波）上传输不同频道，不同的信号利用不同频率传输

      • HFC：混合光纤同轴电缆
      • 下行：30Mbps 上行：2Mbps
        • 各设备通过电缆网络→光纤接入ISP路由器

    • 机构（企业）接入网络：目前，端系统通常直接连接以太网交换机

    • 无限接入网络：

      • 通过共享的无线接入网络连接端系统与路由器
        • 通过基站或称为“接入点”
      • 无线局域网（LANs）：11Mbps，54Mbps
      • 广域无线接入：1、10、100Mbps，如3G，4G，LTE等

  • 网络核心：

    • 功能

      • 路由：确定分组从源到目的传输路径

      • 转发：将分组从路由器的输入端口交换至正确的输出端口

  • Internet结构

    • 端系统通过接入ISP连接到Internet

    • 接入ISP进一步互联，保证两个主机可以互相发送分组

    • 如果每个接入ISP直接彼此互连，O(n^2^)，不适用于大规模网络

    • 如果将每个接入ISP连接到一个国家或全球的ISP，但从商业角度，必定有竞争者，所以这些ISP必须互连。这些ISP可以使用对等链路进行连接，但一般是通过IXP，也就是互联网交换节点来进行连接，而这些节点本身就是一个高速网络

    • 可能出现区域网络连接接入ISP和运营商ISP

    • 内容提供商网络，可能运行其自己的网络，并就近为端用户提供服务、内容

    • 以上这些接口可以称为网络拓扑结构。

网络核心

数据交换

​ 如果将不同主机直接相连，需要的通信链路数为$\frac{N(N-1)}{2}$，且技术上也很难实现。于是现在采用交换设备，每个主机与交换设备相连，每个主机只需要一条链路。但一个交换设备的端口数量有限，且可能与某些主机距离过远，于是现在采用交换设备连接而成的交换网络技术。
